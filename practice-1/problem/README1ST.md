# Practice 1

This is a practice exercise to build a responsive navbar from the starting files.

The files come with pre-existing styles to handle global styles such as colors, sizing, and some padding. You only need to worry about layout styles for the links themselves.

In the first approach, use only CSS grid for the layout styles.

In the second approach, use only flexbox.

Mockup files of how the nav bar should look on mobile and desktop are in the "mockups" subfolder.

## Functional Requirements

Add layout styles to the nav bar styles and make it responsive:

On mobile viewport widths up to 50rem (800px):

- The nav links should be displayed in 1 row, horizontally aligned so space is evenly divided between and around the links.
- On very narrow mobile widths (for example, 320px wide), there should be 10px of space between the links.

On tablet/desktop viewport widths equal to or greater than 50rem (800px):

- The first "Home" link should be horizontally aligned all the way to the left side.
- The other three links should be aligned all the way to the right side, with 20px of space between them.
- Any extra space goes between the first "Home" link and the second "About" link.

## Grid Approach

Build the nav bar using the following files. You shouldn't need to make any changes to the HTML, only the CSS.

- grid.html
- grid.css

## Flexbox Approach

Build the nav bar using the following files. You shouldn't need to make any changes to the HTML, only the CSS.

- flex.html
- flex.css
